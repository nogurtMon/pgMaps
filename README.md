## Self Hosting

**Requirements:** Docker

**1. Install Docker**

- **Mac / Windows** — download and install [Docker Desktop](https://www.docker.com/products/docker-desktop)
- **Linux**
  ```bash
  curl -fsSL https://get.docker.com | sh
  sudo usermod -aG docker $USER && newgrp docker
  ```

**2. Run**

```bash
docker run -d \
  -p 3000:3000 \
  -v postgis_data:/var/lib/postgresql/data \
  -v app_data:/data \
  nogurtmon/postgis-frontend:latest
```

Open `http://localhost:3000`.

Or with Docker Compose — create a `docker-compose.yml`:

```yaml
services:
  app:
    image: nogurtmon/postgis-frontend:latest
    ports:
      - "3000:3000"
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - app_data:/data
    restart: unless-stopped

volumes:
  postgres_data:
  app_data:
```

```bash
docker compose up -d
```

> The app ships with a bundled PostgreSQL database — no external database required for local use.

**Update:**

```bash
docker compose pull && docker compose up -d
```

## Environment Variables

All variables are optional for local self-hosting. Create a `.env` file next to `docker-compose.yml` to set them.

| Variable | Default | Description |
|---|---|---|
| `APP_PASSWORD` | *(none)* | Protects the app with a password. Without it, anyone with the URL can connect databases and read or write data. Public share links (`/share/[id]`) remain accessible regardless. |
| `POSTGRES_URL` | bundled PostgreSQL | Storage database for saved connections, maps, and basemaps. **Required for cloud deployments** (Cloud Run, Railway, Render, etc.) where the container filesystem is ephemeral. Point to a PostgreSQL instance with PostGIS, e.g. Cloud SQL, Neon, or Supabase. |
| `DSN_ENCRYPTION_KEY` | auto-generated | 32-byte hex key used to encrypt saved connection strings. Auto-generated and persisted to the data volume on first run. **Set this explicitly for cloud deployments** — without a persistent volume, a new key is generated on every restart, making saved connections unreadable. Generate one with `openssl rand -hex 32`. |
| `PORT` | `3000` | Port the app listens on. |
| `SHOW_LANDING_PAGE` | *(unset)* | Set to any value to show the marketing landing page at `/`. Otherwise `/` redirects to `/maps`. |
